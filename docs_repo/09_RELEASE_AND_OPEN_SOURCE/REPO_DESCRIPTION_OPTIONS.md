# Repo Description Options

## Goal

These options are written for the GitHub repository description field.

They should be short, clear, and accurate enough to improve search relevance and first-impression quality without overselling the project.

## Recommended Best Option

`Windows desktop app for converting insurance estimate PDFs into structured ESX/XML exports using local OCR and canonical normalization.`

Why this is the best option:

- clear about platform
- clear about domain
- clear about output
- includes useful search terms without sounding inflated

## Description Options

1. `Windows desktop app for converting insurance estimate PDFs into structured ESX/XML exports using local OCR and canonical normalization.`
2. `Open-source Windows app for parsing insurance estimate PDFs and generating ESX/XML outputs with OCR fallback.`
3. `Local-first insurance estimate PDF parser and ESX/XML exporter for Windows, built for messy real-world claim packets.`
4. `Convert insurance estimate PDFs into ESX-style XML packages locally with a Windows desktop app and inspectable Python pipeline.`
5. `Insurance estimate PDF extraction and ESX/XML export tool for Windows, with OCR fallback and deterministic output artifacts.`
6. `Open-source PDF-to-ESX workflow for insurance estimate documents, including OCR, parsing, canonical mapping, and XML export.`
7. `Windows desktop tool for insurance estimate PDF parsing, OCR-assisted extraction, and structured ESX/XML generation.`

## Shorter Technical Option

`Insurance estimate PDF parser and ESX/XML exporter for Windows with OCR fallback.`

## More Business-Facing Option

`Local tool for turning insurance estimate PDFs into structured ESX/XML outputs for property-claims and estimating workflows.`

## More Open-Source Or Community-Facing Option

`Open-source Windows app for insurance estimate PDF parsing, OCR fallback, and ESX/XML export. Contributors welcome.`

## Recommendation

Use the recommended best option unless the maintainer wants to bias the landing page more toward:

- technical contributor discovery
  use the shorter technical option
- business/industry comprehension
  use the business-facing option
- community/open-source tone
  use the community-facing option
