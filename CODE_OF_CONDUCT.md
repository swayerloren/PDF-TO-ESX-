# Code of Conduct

## Our Commitment

We want `PDF TO ESX AGENT` to be a professional, respectful, and technically honest open-source project.

Contributors, maintainers, and users should be able to participate without harassment, hostility, or bad-faith behavior.

## Expected Behavior

- be respectful and direct
- focus criticism on the work, not the person
- share evidence when discussing bugs, layouts, or compatibility claims
- protect sensitive claim, policy, customer, and carrier information
- keep disagreements grounded in code, fixtures, outputs, and documentation

## Unacceptable Behavior

- harassment, threats, or discriminatory language
- doxxing or sharing private personal or claim information
- publishing unredacted customer documents without permission
- deliberate misrepresentation of test evidence, compatibility, or output quality
- bad-faith issue filing, review behavior, or repeated disruption

## Scope

This Code of Conduct applies in:

- GitHub issues
- pull requests
- review discussions
- project documentation
- other public project communication channels managed for this repository

## Reporting

Report Code of Conduct concerns privately to the maintainers through the private contact route described in [SUPPORT.md](SUPPORT.md).

Do not open a public issue for personal-safety concerns or private misconduct reports.

Security-sensitive concerns should follow [SECURITY.md](SECURITY.md) instead.

## Enforcement

Maintainers may remove, edit, or reject comments, commits, code, issues, or other contributions that violate this Code of Conduct.

Maintainers may also apply temporary or permanent participation restrictions when needed to protect the project and its community.
